📡 API AT A GLANCE
═══════════════════════════════════════════════════════════════════════════

ENDPOINTS:
  GET  /health              — Quick liveness check
  GET  /health/ready        — Check API + MongoDB status
  POST /schedule/generate   — Generate schedule (main endpoint)

CORE REQUEST:
  {
    "institution_id": "test-inst-001",
    "term_label": "fall-2024",
    "weights": { ... },              // Optional: customize priorities
    "section_type_durations": { ... }  // Optional: session lengths
  }

CORE RESPONSE:
  {
    "snapshot_id": "uuid",
    "entries": [ ... ],              // Scheduled sessions
    "hard_violations": 0,            // Must be 0
    "soft_penalty": 240.5,           // Lower is better
    "warnings": [ ... ],
    "summary": { ... }
  }

CONSTRAINTS:
  ✅ 9 Hard Constraints (must satisfy)
  ✅ 4 Soft Constraints (optimize with weights)
  ✅ Configurable weights for priority tuning


🚀 QUICK START (FRONTEND)
═══════════════════════════════════════════════════════════════════════════

1. INSTALL
   npm install axios

2. CREATE API CLIENT
   // lib/scheduler-api.ts
   import axios from 'axios';
   const schedulerAPI = axios.create({
     baseURL: process.env.NEXT_PUBLIC_SOLVER_API_URL,
   });
   export default schedulerAPI;

3. CREATE HOOK
   // hooks/useScheduleGeneration.ts
   export function useScheduleGeneration() {
     const [loading, setLoading] = useState(false);
     const generateSchedule = async (institutionId, term) => {
       return await schedulerAPI.post('/schedule/generate', {
         institution_id: institutionId,
         term_label: term,
       });
     };
     return { generateSchedule, loading };
   }

4. USE IN COMPONENT
   const { generateSchedule, loading } = useScheduleGeneration();
   await generateSchedule('test-inst-001', 'fall-2024');

📈 CONSTRAINT SYSTEM (QUICK REFERENCE)
═══════════════════════════════════════════════════════════════════════════

HARD CONSTRAINTS (Must satisfy - cannot be violated):
  H1 - Room No-Overlap          (no double-booking)
  H2 - Staff No-Overlap         (no simultaneous teaching)
  H3 - Room Capacity            (students ≤ capacity)
  H4 - Room Features            (labs, equipment available)
  H5 - Staff Day-Off            (respect time-off)
  H6 - Session Slots            (all slots scheduled)
  H7 - Year-Level Conflicts     (no conflicts in building)
  H8 - Staff Availability       (only during availability)
  H9 - Shared Sections          (same time + room)

SOFT CONSTRAINTS (Optimize with configurable weights):
  S1 - Break Windows            (staff lunch breaks) — weight: 100
  S2 - Consecutive Slots        (teaching blocks) — weight: 80
  S3 - Session Spread           (throughout week) — weight: 60
  S4 - Campus Clustering        (by building) — weight: 40

ADJUST WEIGHTS IN REQUEST:
  Higher weight = higher priority
  Example: Set break_window to 200 to maximize breaks
